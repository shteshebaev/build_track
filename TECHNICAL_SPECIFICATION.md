**Версия:** 1.0
**Дата:** Март 2026
**Статус:** В разработке

---

## 1. Общие сведения

### 1.1 Наименование системы
**Build Track** — комплексная система управления строительными проектами и складским учётом.

### 1.2 Назначение системы
Система предназначена для автоматизации процессов управления строительными объектами, включая:
- Управление проектами и отслеживание прогресса строительства
- Складской учёт материалов
- Управление закупками и поставщиками
- Формирование смет и контроль бюджета
- Учёт продаж недвижимости
- Документооборот и фотоотчёты
- Аналитика и отчётность

### 1.3 Целевая аудитория
- Руководители строительных компаний
- Менеджеры проектов
- Инженеры и прорабы
- Кладовщики и снабженцы
- Бухгалтеры
- Менеджеры по продажам

## 3. Функциональные модули

### 3.1 Дашборд (Главная панель)

**Назначение:** Оперативный мониторинг ключевых показателей деятельности.

**Функциональные возможности:**

| Функция | Описание |
|---------|----------|
| KPI-карточки | 6 ключевых показателей с индикаторами трендов |
| График прогресса | Сравнение планового и фактического прогресса по проектам |
| Распределение статусов | Круговая диаграмма статусов проектов |
| Расход материалов | Столбчатая диаграмма по категориям |
| Лента активности | 5 последних событий с цветовой индикацией |

**KPI-показатели:**
1. Всего проектов (количество + тренд)
2. Стоимость склада (сумма + тренд)
3. Использовано материалов (количество + тренд)
4. Активных заявок (количество + тренд)
5. Прогресс строительства (% + тренд)
6. Выручка от продаж (сумма + тренд)

- Выбор материала

**Инвентаризация:**

### 3.6 Заявки на материалы

**Назначение:** Оформление и согласование заявок на закупку.

#### 3.6.1 Функции модуля

| Функция | Описание |
|---------|----------|
| Список заявок | Номер, проект, заявитель, дата, статус |
| Создание | Мастер создания в 3 шага |
| Статусы | Черновик → На согласовании → Одобрена/Отклонена |
| История | Отслеживание согласований |

#### 3.6.2 Мастер создания заявки

**Шаг 1 — Основная информация:**
- Выбор проекта
- Требуемая дата поставки
- Приоритет (низкий/обычный/высокий/срочный)

**Шаг 2 — Позиции заявки:**
- Добавление материалов
- Указание количества
- Примечания к позициям
- Расчёт итоговой стоимости

**Шаг 3 — Подтверждение:**
- Просмотр сводки
- Общие примечания
- Отправка на согласование

#### 3.6.3 Жизненный цикл заявки

```
Черновик → На согласовании → Одобрена → В доставке → Выполнена
                          ↘ Отклонена
```

## 5. Форматирование данных

### 5.1 Валюта

```
Примеры:
1,234,567 сум → "1.2 млн" (автоформат)
50,200,000,000 сум → "50.2 млрд"
```

**Подсказки:** При наведении показывается полная сумма.

### 5.2 Даты

| Формат | Пример |
|--------|--------|
| Короткий | 15.12.2024 |
| Длинный | 15 декабря 2024 |
| С временем | 15.12.2024, 14:30 |

### 5.3 Числа

- Разделитель тысяч: пробел
- Десятичный разделитель: запятая
- Пример: 1 234 567,89

## 7. Безопасность

### 7.1 Требования к аутентификации

- JWT или OAuth 2.0 токены
- Безопасное хранение токенов
- Автоматический logout при неактивности
- Защита от CSRF-атак
